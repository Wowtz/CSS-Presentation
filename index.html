01 — Hierarquia de Seletores

*, :root, html — Do mais geral ao mais específico

Seletores definem quem vai receber o estilo. A hierarquia vai do mais genérico (*) ao mais específico (ID, inline).

🌐

* — Seletor Universal

Aplica estilos em todos os elementos do documento. Neste projeto, usamos * { box-sizing: border-box; margin: 0; padding: 0; } para padronizar o modelo de caixa e zerar espaçamentos padrão do browser.

🎨

:root — Variáveis CSS

Pseudo-classe que aponta para o elemento raiz do documento (html), mas com especificidade maior. Usada para declarar variáveis CSS (custom properties) acessíveis globalmente, como cores e tipografia.

📄

html / body

Seletores de tipo que estilizam os elementos raiz. Aqui definimos scroll-behavior: smooth no html e a família de fontes e cor de fundo no body. 

/* :root — variáveis disponíveis em TODO o documento */
:root {
  --accent: #e05c2e;
  --bg:     #f4f0e8;
  --text:   #1c1c1e;
}

/* * — aplica em TODOS os elementos */
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

/* html e body — seletores de tipo */
html { scroll-behavior: smooth; }
body {
  background: var(--bg);
  color:      var(--text);
  font-family: 'DM Mono', monospace;
}

Por que usar? Definir variáveis no :root é a base do sistema de design: você muda uma variável e toda a página se atualiza. Isso alimenta o tema escuro desta página — troque o botão acima e veja!

02 — Pseudo-classes

:hover, :nth-child, :link, :root

Pseudo-classes selecionam elementos em um estado específico, sem precisar adicionar classes no HTML.

🖱️

:hover

Ativada quando o cursor está sobre o elemento. Usada aqui em cards, botões do nav, e no demo abaixo. Permite alterar borda, tamanho, cor e sombra de forma acessível.

🔢

:nth-child(n)

Seleciona elementos pela posição entre seus irmãos. Aceita fórmulas como odd, even, 3n. Usado nos pontinhos da timeline, nas bolinhas do demo abaixo e nos dots do "dot-loader".

🔗

:link / :visited

Controlam o estilo de links não visitados e já visitados. Usados no nav para estilizar links antes e depois do clique, garantindo feedback visual claro ao usuário.

a:link    { color: var(--text); }

03 — Combinador Adjacente

O combinador + (Adjacent Sibling)

Seleciona o primeiro elemento imediatamente após outro. Ambos precisam ter o mesmo pai. 

/* h2 + p — o parágrafo LOGO APÓS um h2 */
h2 + p {
  margin-top: .75rem;
  color: var(--text-muted);
}

/* .hover-card:hover + .hover-card — o card adjacente ao hover */
.hover-card:hover + .hover-card {
  transform: scale(1.03);
  border-color: color-mix(in srgb, var(--accent2) 50%, transparent);
}

/* Underline animado: link + ::after com transition */
.nav-links a::after {
  content: '';
  position: absolute;
  bottom: -3px;
  left: 0; right: 100%;
  height: 2px;
  background: var(--accent);
  transition: right .35s ease;
}
.nav-links a:hover::after { right: 0; }

Nesta página o combinador + é usado em pelo menos 5 lugares: h2 + p, h3 + p, h4 + p, .hover-card:hover + .hover-card, e o .card::before animado. Abra o DevTools e inspecione!

04 — backdrop-filter

Efeito "vidro fosco" com backdrop-filter

Aplica filtros gráficos (blur, saturação, brilho) no conteúdo atrás do elemento. Funciona como o efeito vidro do iOS/macOS.

🧪 Demo — blur(12px) + saturate(180%)

🎨🌈✨🎭
backdrop-filter: blur(12px) saturate(180%)
Este painel tem background: rgba(255,255,255,.12) e o blur é aplicado no fundo atrás dele — não no conteúdo interno. 
/* A navegação desta página usa backdrop-filter */
nav {
  backdrop-filter: blur(18px) saturate(160%);
  -webkit-backdrop-filter: blur(18px) saturate(160%);
  background: rgba(0,0,0,.18);
}

/* O painel do demo acima */
.backdrop-panel {
  backdrop-filter: blur(12px) saturate(180%);
  background: rgba(255,255,255,.12);
  border: 1px solid rgba(255,255,255,.2);
}

Quando usar? Navbars fixas, modais, sidebars, tooltips — qualquer elemento flutuante onde você quer manter legibilidade sem bloquear completamente o fundo. É onipresente no modo escuro de apps modernos.

05 — História

Quando nasceram esses recursos?

CSS existe desde 1996, mas muitos seletores e propriedades vieram muito depois.

1996

CSS1 — Os primeiros seletores

Seletores de tipo (html, body, p), seletor de classe (.classe) e ID (#id). Também chegaram :link e :visited. O combinador adjacente + já estava na especificação inicial.

1998

CSS2 — :hover, :focus, :active, seletor universal

Chegou o * (seletor universal), o :hover, :focus e :active. O CSS2 formalizou a especificidade e a cascata como as conhecemos hoje.

2011

CSS3 — :nth-child, :root, variáveis (rascunho)

O CSS Selectors Level 3 trouxe :nth-child(n), :nth-of-type, :root, :not(), e dezenas de outras pseudo-classes. O rascunho de Custom Properties (variáveis) data de 2012.

2015–2017

CSS Custom Properties — suporte amplo em browsers

As variáveis CSS (--minha-var no :root) chegaram ao Chrome 49 (2016), Firefox 31 (2014) e Safari 9.1 (2016). Hoje têm suporte universal. Isso revolucionou temas e design systems.

2019–hoje

backdrop-filter — suporte estável

Especificado no CSS Filter Effects Level 2. Chegou ao Safari em 2015, mas ficou por anos atrás de flag no Chrome. Suporte estável no Chrome 76 (2019) e Firefox 103 (2022). É uma das propriedades que mais cresceu em uso nos últimos anos.

06 — SCSS vs Sass

São diferentes — mas da mesma família

Sass (Syntactically Awesome Style Sheets) é o pré-processador. SCSS é a sintaxe mais nova e popular dele. São dialetos do mesmo compilador.

💅 SCSS (recomendado)
  • Usa { } e ; — idêntico ao CSS normal
  • Todo CSS válido é SCSS válido
  • Extensão: .scss
  • Suporta nesting, variáveis $var, mixins, funções
  • Mais fácil de adotar em projetos existentes
  • Usado em Angular, Bootstrap, Material UI
✏️ Sass (sintaxe indentada)
  • Usa indentação, sem { } ou ;
  • Sintaxe mais compacta e minimalista
  • Extensão: .sass
  • Mesmos recursos do SCSS
  • Mais difícil de migrar a partir de CSS puro
  • Menos popular hoje em dia
/* ── SCSS (.scss) ───────────────────────────── */
$accent: #e05c2e;

.card {
  background: white;

  &:hover {
    transform: translateY(-5px);
    color: $accent;
  }

  h3 + p {
    color: gray;
  }
}

/* ── Sass (.sass) — mesma coisa, outra sintaxe ── */
$accent: #e05c2e

.card
  background: white

  &:hover
    transform: translateY(-5px)
    color: $accent

  h3 + p
    color: gray

Por que usar? Ambos compilam para CSS normal — o browser nunca vê o SCSS. O ganho é a produtividade: nesting, mixins, funções e variáveis ($var) que existiam antes das variáveis nativas do CSS. Hoje as CSS Custom Properties cobrem muito do que as variáveis Sass faziam, mas o SCSS ainda é rei para nesting e lógica reutilizável.

07 — Onde esses conceitos aparecem

No mundo real, você encontra em todo lugar

Pseudo-classes, combinadores e variáveis CSS são base de praticamente qualquer interface moderna.

🌙

Dark Mode

Mudar :root ou usar @media (prefers-color-scheme: dark) para trocar variáveis instantaneamente.

🧭

Navbars & Headers

backdrop-filter, :hover com underline animado via combinador +.

📋

Tabelas & Listas

Zebra striping com tr:nth-child(even). Destaque de linhas com :hover.

🃏

Cards & Grids

:hover com transform e box-shadow para elevação. :nth-child para espaçamento alternado.

📝

Formulários

:focus, :valid, :invalid e :placeholder-shown para feedback ao usuário.

🖼️

Modais & Sidebars

backdrop-filter no overlay, :root para desabilitar scroll com variável de estado.

🎨

Design Systems

Bootstrap, Tailwind, Material UI — todos usam :root e :hover como fundação do sistema de tokens.

♿

Acessibilidade

:focus-visible mostra foco apenas para navegação por teclado sem afetar usuários de mouse.

08 — Dark Mode na prática

Como o tema escuro funciona nesta página

O segredo está nos CSS Custom Properties no :root. Trocar o tema é só adicionar/remover uma classe no html e sobrescrever as variáveis.

styles.css — comparativo de temas
☀️ Tema Claro

Background #f4f0e8, texto escuro, accent laranja.

:root (padrão)
🌙 Tema Escuro

Background #0d0d14, texto claro, accent laranja intenso.

:root.dark
/* Tema padrão (claro) */
:root {
  --bg:     #f4f0e8;
  --text:   #1c1c1e;
  --accent: #e05c2e;
}

/* Dark mode — sobrescreve as mesmas variáveis */
:root.dark {
  --bg:     #0d0d14;
  --text:   #e8e8f0;
  --accent: #ff6b35;
}

/* Também pode usar media query */
@media (prefers-color-scheme: dark) {
  :root {
    --bg:   #0d0d14;
    --text: #e8e8f0;
  }
}
// Trocar tema apenas manipulando classe no html
document.querySelectorAll('.theme-btn').forEach(btn => {
  btn.addEventListener('click', () => {
    const theme = btn.dataset.theme;

    // Remove todos os temas anteriores
    document.documentElement.classList
      .remove('dark', 'green', 'purple');

    // Adiciona o novo (exceto 'light' que é o padrão)
    if (theme !== 'light') {
      document.documentElement.classList.add(theme);
    }
  });
});

Por que isso é poderoso? O body, nav, cards e todos os elementos usam var(--bg), var(--text) etc. Quando você troca a classe no :root, toda a página muda com uma única operação, sem tocar no HTML nem precisar de JavaScript complexo.
